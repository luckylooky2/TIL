## 1. docker
### grafana.ini

```ini
app_mode = production
instance_name = ${HOSTNAME}

#################################### Server ####################################
[server]
protocol = http
http_addr =
http_port = 3000

#################################### Database ####################################
[database]
type = mysql
host = db:3306
name = grafana
user = grafana
password = grafana


#################################### Logging ##########################
[log]
mode = console
level = info

#################################### Alerting ############################
[alerting]
enabled = true
```
- mysql을 외부 db로 변경 및 분리
- https://velog.io/@ghdud0503/Docker-%EA%B8%B0%EC%B4%88-17-%EB%8F%84%EC%BB%A4-%EC%BB%B4%ED%8F%AC%EC%A6%88-Grafana-MySQL-%EA%B5%AC%EC%B6%95

### docker-compose.yaml

```yaml
version: "3.9"

services:
  db:
    image: mysql:latest
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: grafana
      MYSQL_DATABASE: grafana
      MYSQL_USER: grafana
      MYSQL_PASSWORD: grafana
    volumes:
      - mysql-data:/var/lib/mysql
    logging:
      driver: "json-file"
      options:
        max-size: "8m"
        max-file: "10"

  grafana:
    depends_on:
      - db
    image: chanhyle/grafana-oss:dev-ubuntu
    restart: unless-stopped
    ports:
      - 3000:3000
    volumes:
      - ./files/grafana.ini:/etc/grafana/grafana.ini:ro
      - grafana-data:/var/lib/grafana
    logging:
      driver: "json-file"
      options:
        max-size: "8m"
        max-file: "10"

volumes:
  mysql-data: {}
  grafana-data: {}
```
- mysql 컨테이너를 추가하고, mysql 컨테이너는 local volume(`mysql-data`)에 마운트
- grafana 컨테이너는 mysql 컨테이너와 db 연결

## 2. k8s
### grafana-deployment.yaml

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: grafana-provisioning-datasource
  labels:
    app: grafana
data:
  grafana.ini: |-
    app_mode = production
    instance_name = ${HOSTNAME}

    #################################### Server ####################################
    [server]
    protocol = http
    http_addr =
    http_port = 3000

    #################################### Database ####################################
    [database]
    type = mysql
    host = mysql-service.default.svc.cluster.local:3306
    name = grafana
    user = grafana
    password = grafana


    #################################### Logging ##########################
    [log]
    mode = console
    level = info

    #################################### Alerting ############################
    [alerting]
    enabled = true
  datasource-sample.yaml: |-
    apiVersion: 1

    deleteDatasources:
    - name: prometheus

    datasources:
    - name: prometheus
      type: prometheus
      access: proxy
      url: http://prom-kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090
      isDefault: true
      version: 1
      editable: false
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: grafana-deployment
  labels:
    app: grafana
spec:
  replicas: 1
  selector:
    matchLabels:
      app: grafana
  template:
    metadata:
      labels:
        app: grafana
    spec:
      containers:
      - name: grafana
        image: grafana/grafana:latest
        env:
        - name: name
          value: value
        ports:
        - containerPort: 3000
        volumeMounts:
        - name: grafana-provisioning-datasource
          mountPath: /etc/grafana/grafana.ini
          subPath: grafana.ini
        - name: grafana-provisioning-datasource
          mountPath: /etc/grafana/provisioning/datasources/datasource-sample.yaml
          subPath: datasource-sample.yaml
      volumes:
      - name: grafana-provisioning-datasource
        configMap:
          name: grafana-provisioning-datasource
---
apiVersion: v1
kind: Service
metadata:
  name: grafana-service
spec:
  selector:
    app: grafana
  ports:
    - protocol: TCP
      name: grafana-port
      port: 3000
      targetPort: 3000
  type: NodePort
```

```shell
k delete deploy grafana-deployment
k delete cm grafana-provisioning-datasource

k create -f grafana-deployment.yaml
```
- configmap에 `grafana.ini` 추가
- `grafana.ini`
	- mysql-service.default.svc.cluster.local:3306 로 mysql pod와 db 연결
### mysql-deployment.yaml

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mysql
spec:
  selector:
    matchLabels:
      app: mysql
  strategy:
    type: Recreate
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - image: mysql/mysql-server:latest
        name: mysql
        env:
        - name: MYSQL_USER
          value: grafana
        - name: MYSQL_PASSWORD
          value: grafana
        - name: MYSQL_ROOT_PASSWORD
          value: grafana
        - name: MYSQL_DATABASE
          value: grafana
        ports:
        - containerPort: 3306
          name: mysql
---
apiVersion: v1
kind: Service
metadata:
  name: mysql-service
  namespace: default
spec:
  selector:
    app: mysql
  ports:
    - protocol: TCP
      name: mysql-port
      port: 3306
      targetPort: 3306
  type: NodePort
```

```shell
k delete deploy mysql-deployment

k create -f mysql-deployment.yaml
```
- env에 `grafana.ini`에 입력한 환경변수와 같은 값으로 추가
